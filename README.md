# AIMC 数字人主持系统

## 项目概述

AIMC（AI主持系统）是一套基于POMASA多智能体架构的数字人主持系统，专为大会双主持场景设计。系统实现了与人类主持人的协同主持功能，支持智能语音识别、上下文理解、临场应变和演讲评价等核心能力。

## 核心功能

### P0 - 基础主持功能
- 智能语音检测：准确判断人类主持人何时结束发言
- 流程控制：按预定主持稿流程推进
- 无缝衔接：数字人自然地开始和结束发言

### P1 - 临场应变
- 上下文分析：理解用户的即兴发言
- 衔接生成：生成自然的过渡衔接词
- 流程恢复：回到预定的主持流程

### P2 - 临场互动
- 技能集成：调用外部skills进行即兴发挥
- 快速响应：无需深度思考，即时回应

### P3 - 演讲评价
- 要点记录：实时记录嘉宾演讲要点
- 评价生成：生成专业的评价内容
- 预制开场：使用预制的评价开场白

## 技术架构

### 基于POMASA的三层架构

```
AIMC/
├── agents/                          # Agent蓝图文件
├── methodology/                     # 方法论指导
├── materials/                       # 原始材料
├── drafts/                          # 临时状态
└── final/                          # 最终产出
```

### 核心智能体

1. **Orchestrator** - 主持流程协调器
2. **HumanDetector** - 用户语音检测
3. **AILineDeliver** - 数字人台词交付
4. **ContextAnalyzer** - 上下文分析
5. **ResponseGenerator** - 应变响应生成
6. **SpeechEvaluator** - 演讲评价

## 开始使用

### 前置要求

- Python 3.8+
- 可用的TTS API（如AWS Polly、Azure TTS）
- 可用的语音识别API
- Seedance数字人素材（可选）

### 快速开始

1. 克隆项目到本地
2. 配置API密钥和参数
3. 准备主持稿和素材
4. 启动系统

## 开发指南

### 创建新的Agent

1. 在`agents/`目录下创建新的Blueprint文件
2. 遵循COR-01 Prompt-Defined Agent模式
3. 包含角色定位、行为规范、输出要求等部分

### 添加新的技能

1. 在`materials/skills/`目录下添加技能配置
2. 更新ResponseGenerator以支持新技能
3. 测试技能集成效果
