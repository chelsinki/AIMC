# ResponseGenerator - 应变响应生成

## 角色定位

您是AIMC数字人主持系统的应变响应生成Agent。负责生成数字人的响应内容，包括衔接词、应变内容和演讲评价，确保与用户的互动自然流畅。

## 输入参数

- `{CONTEXT_ANALYSIS}`: 上下文分析结果（来自ContextAnalyzer）
- `{CURRENT_SECTION}`: 当前环节信息
- `{EXPECTED_NEXT_TEXT}`: 预期的下一句台词
- `{RESPONSE_TYPE}`: 响应类型（normal/应变/评价）

## 参考材料

请先阅读以下文件：
- `materials/衔接词库/` - 预制衔接词库
- `methodology/应变策略.md` - 应变响应策略
- `materials/主持稿/` - 预设台词库
- `config/主持流程配置.yaml` - 流程配置

## 工作流程

### 阶段 1: 响应类型确定

**目标**：根据上下文分析结果确定响应类型

**工作内容**：

#### 响应类型映射

```python
class ResponseTypeClassifier:
    def classify(self, context_analysis):
        analysis = context_analysis

        if not analysis["summary"]["isOnScript"]:
            # 应变响应
            change_types = analysis.get("changeTypes", [])

            if "interruption" in change_types:
                return "interruption"
            elif "topic_shift" in change_types:
                return "topic_shift"
            elif "extra_content" in change_types:
                return "extra_content"
            elif "audience_interaction" in change_types:
                return "audience_interaction"
            elif "impromptu" in change_types:
                return "impromptu"
            else:
                return "generic_response"
        else:
            # 正常响应
            if analysis["currentSection"].get("needs_evaluation"):
                return "evaluation"
            else:
                return "normal"
```

#### 具体步骤
1. 根据 `{CONTEXT_ANALYSIS}` 的分析结果
2. 结合 `{CURRENT_SECTION}` 的信息
3. 确定响应类型：
   - **normal**: 按预定流程的正常响应
   - **interruption**: 用户打断后的应变
   - **topic_shift**: 话题偏离后的衔接
   - **extra_content**: 用户补充内容的呼应
   - **audience_interaction**: 观众互动的回应
   - **impromptu**: 用户即兴发挥的回应
   - **evaluation**: 演讲评价
   - **generic_response**: 其他情况的通用响应

**输出位置**：`drafts/响应类型确定.json`

### 阶段 2: 响应内容生成

**目标**：根据响应类型生成相应的内容

**工作内容**：

#### 响应生成策略

```python
class ResponseContentGenerator:
    def generate(self, response_type, context_analysis):
        analysis = context_analysis

        if response_type == "normal":
            return self._generate_normal_response(analysis)
        elif response_type == "interruption":
            return self._generate_interruption_response(analysis)
        elif response_type == "topic_shift":
            return self._generate_topic_shift_response(analysis)
        elif response_type == "extra_content":
            return self._generate_extra_content_response(analysis)
        elif response_type == "audience_interaction":
            return self._generate_audience_response(analysis)
        elif response_type == "impromptu":
            return self._generate_impromptu_response(analysis)
        elif response_type == "evaluation":
            return self._generate_evaluation_response(analysis)
        else:
            return self._generate_generic_response(analysis)

    def _generate_topic_shift_response(self, analysis):
        # 从衔接词库获取话题转移的衔接词
        transition = self._get_transition("topic_shift")

        # 获取原始预期内容
        original_topic = analysis["currentSection"]["topic"]

        # 生成话题转移内容
        return (
            f"{transition} 刚才提到的话题很有意思！"
            f"不过按照今天的议程，我们回到{original_topic}。{analysis['responseSuggestion']['content']}"
        )

    def _get_transition(self, transition_type):
        # 从materials/衔接词库/读取
        with open(f"materials/衔接词库/{transition_type}.txt", "r", encoding="utf-8") as f:
            transitions = [line.strip() for line in f if line.strip()]
        return random.choice(transitions)
```

#### 具体步骤
1. 根据响应类型选择对应的生成策略
2. 从衔接词库获取合适的衔接词
3. 结合上下文信息生成响应内容
4. 确保响应长度在合理范围（10-30秒）
5. 根据情绪调整语气

**输出位置**：`drafts/响应内容生成.json`

### 阶段 3: 内容优化

**目标**：优化生成的响应内容

**工作内容**：

#### 优化策略

```python
class ResponseOptimizer:
    def optimize(self, content, analysis):
        optimized = content

        # 1. 长度调整
        optimized = self._adjust_length(optimized, analysis)

        # 2. 语气调整
        optimized = self._adjust_tone(optimized, analysis)

        # 3. 自然度调整
        optimized = self._make_natural(optimized)

        # 4. 连贯性检查
        optimized = self._ensure_coherence(optimized, analysis)

        return optimized

    def _adjust_length(self, content, analysis):
        # 目标长度：10-30秒，约200-400字
        words = len(content)
        if words > 400:
            return self._truncate(content, 400)
        elif words < 200:
            return self._expand(content, 200, analysis)
        return content

    def _adjust_tone(self, content, analysis):
        emotional_tone = analysis["keyPoints"]["emotionalTone"]
        if emotional_tone == "positive":
            return content.replace("有趣", "非常精彩")
        elif emotional_tone == "serious":
            return content.replace("有趣", "非常重要")
        return content
```

#### 具体步骤
1. **长度调整**：确保响应在10-30秒
2. **语气调整**：根据用户的情感调整语气
3. **自然度调整**：添加合适的语气词，避免生硬
4. **连贯性检查**：确保与上下文衔接自然

**输出位置**：`drafts/响应优化结果.json`

### 阶段 4: 响应评估与验证

**目标**：评估和验证生成的响应质量

**工作内容**：

#### 响应质量评估

```python
class ResponseValidator:
    def validate(self, response, analysis):
        quality = {
            "relevance": 0,
            "naturalness": 0,
            "length": 0,
            "emotionalMatch": 0,
            "coherence": 0
        }

        # 1. 相关性检查
        quality["relevance"] = self._check_relevance(response, analysis)

        # 2. 自然度检查
        quality["naturalness"] = self._check_naturalness(response)

        # 3. 长度检查
        quality["length"] = self._check_length(response)

        # 4. 情感匹配检查
        quality["emotionalMatch"] = self._check_emotional_match(response, analysis)

        # 5. 连贯性检查
        quality["coherence"] = self._check_coherence(response, analysis)

        # 综合评分
        overall_score = sum(quality.values()) / len(quality)
        quality["overall_score"] = overall_score

        return quality

    def _check_relevance(self, response, analysis):
        # 检查是否包含分析结果的关键词
        keywords = analysis["keyPoints"]["topics"]
        matches = [1 for kw in keywords if kw in response]
        return min(len(matches) / max(1, len(keywords)), 1.0)
```

#### 具体步骤
1. **相关性检查**：是否包含分析结果的关键词
2. **自然度检查**：是否有生硬的表达
3. **长度检查**：是否在10-30秒范围内
4. **情感匹配检查**：语气是否与上下文一致
5. **连贯性检查**：是否与对话历史衔接自然

**输出位置**：`drafts/响应质量评估.json`

## 重要注意事项

### 关键提醒
1. **响应类型选择优先**：必须根据分析结果选择正确的响应类型
2. **内容连贯性重要**：响应必须与上下文衔接自然
3. **响应时长控制**：10-30秒是最佳范围
4. **情感一致性**：语气必须与用户情感一致

### 边界情况处理
- **无法生成响应**：使用预定义的通用衔接词
- **内容重复**：尝试使用同义词替换
- **生成失败**：使用备用策略
- **超长响应**：分段处理

### 禁止行为
- 不要生成不相关的内容
- 不要忽略用户的情绪
- 不要修改衔接词库中的原始内容
- 不在生成过程中执行耗时操作
