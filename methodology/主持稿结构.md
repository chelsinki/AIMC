# 主持稿结构规范

## 概述

本文档定义AIMC数字人主持系统中主持稿的格式规范和结构要求。

## 文件格式

主持稿采用YAML格式，文件名建议使用：`{环节名}.yaml` 或 `{序号}_{环节名}.yaml`

## 基本结构

```yaml
环节信息:
  名称: 开场词
  序号: 1
  预期时长: 120
  预期发言顺序:
    - human
    - ai
    - human

主持人台词:
  human:
    文本: "大家好，欢迎来到今天的活动！"
    预期时长: 30
    关键词:
      - 欢迎
      - 活动
    语气: 热情
    衔接提示: 说完后稍作停顿，等待AI开始

  ai:
    文本: "感谢主持人的精彩开场！今天我们有幸邀请到..."
    预期时长: 90
    关键词:
      - 感谢
      - 邀请
    语气: 亲切
    表情: 微笑
    衔接提示: 说完后看向主持人，等待下一句

  human_next:
    文本: "是的，今天的嘉宾都非常重量级..."
    预期时长: 60
    关键词:
      - 嘉宾
      - 重量级

衔接词库:
  正常衔接:
    - "好的，接下来..."
    - "是的，我们继续..."
    - "非常好，现在..."

  打断衔接:
    - "抱歉打断一下，"
    - "等等，刚才提到的..."
    - "不好意思，我想补充..."

  跳过衔接:
    - "好的，这个话题我们后面再讨论，"
    - "时间关系，我们先进入下一环节，"
    - "这个很有意思，不过我们先..."

  超时提示:
    - "我们时间差不多了，"
    - "好的，感谢分享，"
    - "非常精彩，我们..."

配置:
  允许即兴: true
  最大即兴时长: 30
  超时阈值: 1.3
  静默判断时长: 2.0
  语音检测阈值: 500
```

## 字段说明

### 环节信息

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 名称 | string | 是 | 环节名称，用于显示和日志 |
| 序号 | int | 是 | 环节序号，用于排序 |
| 预期时长 | int | 是 | 总预期时长（秒） |
| 预期发言顺序 | array | 是 | 预期的发言顺序 |

### 主持人台词

#### 通用字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 文本 | string | 是 | 台词文本内容 |
| 预期时长 | int | 是 | 预期发言时长（秒） |
| 关键词 | array | 否 | 关键词列表，用于相似度检测 |
| 语气 | string | 否 | 预期语气（热情/亲切/严肃/中性） |

#### human 特有字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 衔接提示 | string | 否 | 给主持人的提示 |

#### ai 特有字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 表情 | string | 否 | 数字人表情（平静/微笑/兴奋/严肃） |
| 衔接提示 | string | 否 | 给AI的提示 |

### 衔接词库

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 正常衔接 | array | 否 | 正常流程衔接词 |
| 打断衔接 | array | 否 | 用户打断时的衔接词 |
| 跳过衔接 | array | 否 | 跳过时的衔接词 |
| 超时提示 | array | 否 | 超时时的提示词 |

### 配置

| 字段 | 类型 | 默认 | 说明 |
|------|------|------|------|
| 允许即兴 | boolean | true | 是否允许即兴发挥 |
| 最大即兴时长 | int | 30 | 最大即兴时长（秒） |
| 超时阈值 | float | 1.3 | 超时阈值（倍数） |
| 静默判断时长 | float | 2.0 | 静默判断时长（秒） |
| 语音检测阈值 | int | 500 | 语音检测阈值 |

## 多语言支持

如果需要支持多语言，可以使用以下结构：

```yaml
主持人台词:
  human:
    文本:
      zh: "大家好，欢迎来到今天的活动！"
      en: "Hello everyone, welcome to today's event!"
    预期时长: 30
    # 其他字段...
```

## 验证检查清单

- [ ] 所有必填字段已填写
- [ ] 发言顺序合理
- [ ] 预期时长总和合理
- [ ] 关键词有代表性
- [ ] 衔接词库有足够选择
- [ ] YAML格式正确，可解析
- [ ] 文本内容符合场景

## 最佳实践

1. **关键词选择**：选择3-5个最核心的关键词
2. **预期时长**：根据实际演讲者语速调整
3. **衔接词**：准备多个版本，增加多样性
4. **语气提示**：明确标注，帮助AI理解
5. **表情提示**：适合AI数字人的表情范围
